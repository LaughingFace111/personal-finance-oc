# 记账1 - 交易页面操作逻辑详解

## 一、入口与导航

### 1.1 进入交易列表页
- **路径**: 侧边栏菜单 → 「交易」选项
- **URL**: `/transactions`
- **侧边栏菜单项** (左侧 Drawer 展开):
  ```
  - 首页 (/dashboard)
  - 交易 (/transactions)  ← 当前页
  - 账户 (/accounts)
  - 分类 (/categories)
  - 标签 (/tags)
  - 贷款 (/loans)
  - 导入 (/imports)
  - 报表 (/reports)
  - 设置 (/settings)
  ```

### 1.2 新增交易入口
进入交易列表页面后，可以通过以下方式新增交易：

1. **点击「记一笔」按钮** (空状态时显示)
   - 位置：交易列表为空时，页面中央显示 Empty 组件，附带「记一笔」按钮
   - 点击后跳转到 `/transactions/new`

2. **点击 FAB 浮动按钮** (右下角)
   - 页面右下角有一个 FAB (Floating Action Button)
   - 点击展开三个选项：
     - 「收入/支出」→ 跳转 `/add-transaction`
     - 「转账」→ 跳转 `/transfer`
     - 「其他」→ 跳转 `/other`
   
3. **侧边栏新增**
   - 账户页面可添加账户
   - 分类页面可添加分类
   - 标签页面可添加标签

## 二、交易列表页 (TransactionsPage)

**URL**: `/transactions`

### 2.1 页面布局

#### 顶部区域
- **标签筛选器** (仅在非选择模式下显示)
  - 位置：页面顶部
  - 组件：Ant Design `Select` 组件
  - 功能：按标签筛选交易记录
  - 显示逻辑：显示所有标签，下拉选项中展示为「父标签 > 子标签」格式（如果是二级标签）
  - 清空按钮：允许清除筛选条件

- **顶部操作栏** (根据选择模式有两种状态)

**模式1：普通模式** (selectionMode === 'none')
- 左侧：显示交易记录数量，如「12 条记录」
- 右侧：两个小按钮
  - 「退款」按钮 → 进入选择模式，选择 refund
  - 「删除」按钮 → 进入选择模式，选择 delete

**模式2：选择模式** (selectionMode === 'delete' || 'refund')
- 左侧：复选框 + 已选数量，如「已选 3 项」
- 右侧：
  - 「取消」按钮 → 退出选择模式
  - 「确认删除」按钮 (删除模式下) → 执行批量删除
  - 「确认退款」按钮 (退款模式下) → 执行单笔退款

### 2.2 交易列表展示

#### 列表项结构 (每条交易记录)
每条交易以 `List.Item` 形式展示，包含以下信息：

**左侧信息区** (flex: 1)
1. **第一层：二级分类名称** (最醒目)
   - 字体：15px，加粗
   - 颜色：`var(--text-primary)`
   - 备选显示：如果没有分类，显示商户名称 (merchant) 或 "-"

2. **第二层：备注** (浅色小字)
   - 字体：13px
   - 颜色：`var(--text-secondary)`
   - 样式：单行省略 (overflow: hidden, text-overflow: ellipsis, white-space: nowrap)
   - 仅当备注存在时显示

3. **第三层：标签区** (紧凑展示)
   - 显示逻辑：最多显示 2 个标签
   - 格式：Tag 组件，颜色继承自标签配置
   - 超出提示：如果超过 2 个标签，显示 "+N" 如 "+3"

**右侧金额区** (flexShrink: 0)
1. **金额**
   - 字体：16px，加粗
   - 颜色根据交易方向：
     - 收入 (direction === 'in'): 绿色 `#52c41a`，前缀 "+"
     - 退款 (direction === 'refund'): 蓝色 `#1890ff`，前缀 "↩"
     - 支出 (direction === 'out'): 红色 `#ff4d4f`，前缀 "-"
   - 格式：¥ + 金额（保留两位小数）

2. **账户名称**
   - 字体：12px
   - 颜色：`var(--text-secondary)`
   - 样式：单行省略，最大宽度 100px

### 2.3 交互操作

#### 单击交易项
- **普通模式下**：点击任意交易项 → 跳转到编辑页 `/transactions/{id}`
- **选择模式下**：点击交易项 → 切换该条的选中状态

#### 复选框操作 (选择模式下)
- 点击复选框 → 切换该条的选中状态
- 顶部的全选复选框：
  - 点击 → 切换全选/取消全选
  - 中间状态 (indeterminate)：当已选数量 > 0 且 < 总数时显示

## 三、交易编辑/新增页面 (TransactionFormPage)

**URL**: 
- 新增：`/transactions/new`
- 编辑：`/transactions/{id}`

### 3.1 页面结构

#### 顶部导航栏
- **左侧**：返回按钮 ←，点击返回交易列表 `/transactions`
- **中间**：支出/收入切换按钮
  - 两个圆形按钮组成，背景色随选中状态变化
  - 「支出」选中时背景红色 「收入」选中时背景绿色
  - 点击切换时清空错误提示
- **右侧**：占位宽度 (48px)

#### 金额输入区
- **位置**：顶部导航栏下方
- **背景**：渐变色 (根据支出/收入类型选择红色系/绿色系)
- **显示内容**：
  - 标签：「支出金额」或「收入金额」(14px, #999)
  - 输入框：金额输入 (40px, 加粗)
  - 前缀：「¥」符号 (28px)
  - 边框：错误时显示对应颜色的边框

#### 表单区域
使用 `div` + 卡片样式，包含以下字段：

**1. 账户选择**
- 标签：「账户」
- 组件：`Select` 下拉框
- 必填：是（不选会显示红色错误提示）
- 占位：「选择账户」
- 选项：显示所有账户名称

**2. 分类选择**
- 标签：「分类」
- 组件：`Select` 下拉框
- 必填：否
- 占位：「选择分类（可选）」
- 选项过滤：仅显示与当前类型（支出/收入）匹配的分类
- 选项格式：显示分类名称，如果是二级分类显示父分类图标

**3. 日期选择**
- 标签：「日期」
- 组件：`input type="date"`
- 默认值：当天日期 (YYYY-MM-DD)
- 样式：圆角输入框

**4. 标签选择**
- 标签：「标签」
- 组件：`Select` 多选框
- 必填：否
- 占位：「选择标签（可选）」
- 最多显示：3 个标签 (maxTagCount={3})
- 选项分组：按父标签分组显示，展示为「父标签 > 子标签」格式
- 颜色：子标签继承父标签的颜色

**5. 备注输入**
- 标签：「备注（可选）」
- 组件：`input` 文本输入
- 占位：「添加备注」

### 3.2 表单验证规则

| 字段 | 验证规则 | 错误提示 |
|------|----------|----------|
| 金额 | 必须填写且 > 0 | 「请输入金额」 |
| 账户 | 必须选择 | 「请选择账户」 |
| 分类 | 可选 | - |
| 日期 | 可选，默认当天 | - |
| 标签 | 可选 | - |
| 备注 | 可选 | - |

### 3.3 提交操作

#### 按钮区域
- **取消按钮**
  - 位置：左侧
  - 样式：空心按钮，高度 48px，圆角 12px
  - 点击 → 返回交易列表 `/transactions`

- **提交按钮**
  - 位置：右侧
  - 样式：实心按钮，高度 48px，圆角 12px，颜色与当前类型匹配（红/绿）
  - 文字：
    - 新增模式：「记一笔」
    - 编辑模式：「保存修改」
  - 点击 → 执行提交逻辑

#### 提交逻辑
1. **验证表单**：检查金额和账户是否有效
2. **构造请求体**：
   ```json
   {
     "transaction_type": "income" | "expense",
     "amount": 数值,
     "direction": "in" | "out",
     "account_id": "账户ID",
     "category_id": "分类ID" | null,
     "note": "备注内容",
     "occurred_at": "ISO日期字符串",
     "book_id": "账本ID",
     "tags": "JSON字符串数组" | null
   }
   ```
3. **发送请求**：
   - 新增：POST `/api/transactions`
   - 编辑：PATCH `/api/transactions/{id}`
4. **处理响应**：
   - 成功：显示成功提示 → 跳转回交易列表
   - 失败：显示错误提示

## 四、批量操作

### 4.1 删除操作

#### 进入方式
1. 在交易列表页点击「删除」按钮
2. 进入选择模式 (selectionMode === 'delete')

#### 操作流程
1. **选择模式**：顶部显示已选数量，复选框可见
2. **选择交易**：点击复选框或直接点击交易项切换选中状态
3. **全选**：点击顶部复选框可全选/取消
4. **确认删除**：
   - 点击「确认删除」按钮
   - 弹出确认框：「确定要删除选中的 X 条交易吗？」
   - 确认后逐条调用 DELETE `/api/transactions/{id}`
5. **结果**：删除成功后刷新列表，取消选择模式

#### 限制
- 无单笔/批量限制，可任意选择多条

### 4.2 退款操作

#### 进入方式
1. 在交易列表页点击「退款」按钮
2. 进入选择模式 (selectionMode === 'refund')

#### 操作流程
1. **选择模式**：顶部显示已选数量，复选框可见
2. **选择交易**：点击复选框选择单条交易
3. **限制检查**：
   - 批量选择时提示：「退款暂只支持单笔操作，请只选择一条记录」
   - 只能对支出交易 (direction === 'out') 发起退款
4. **确认退款**：
   - 点击「确认退款」按钮
   - 弹出确认框，显示退款金额
   - 确认后调用 POST `/api/transactions/refund`
5. **结果**：退款成功后刷新列表，取消选择模式

#### 退款请求体
```json
{
  "book_id": "账本ID",
  "original_transaction_id": "原交易ID",
  "refund_account_id": "退款账户ID",
  "amount": "退款金额",
  "occurred_at": "退款时间（ISO格式）"
}
```

## 五、其他新增交易方式

### 5.1 收入/支出页面 (AddTransactionPage)

**URL**: `/add-transaction`

- 使用与 TransactionFormPage 相同的表单逻辑
- 专门用于快速记一笔
- 不涉及编辑已有交易

### 5.2 转账页面 (TransferPage)

**URL**: `/transfer`

- 用于账户间转账
- 涉及源账户和目标账户
- 不属于交易列表的直接编辑

### 5.3 其他交易页面 (OtherTransactionPage)

**URL**: `/other`

- 用于特殊类型的交易记录
- 与标准交易表单类似

## 六、数据加载逻辑

### 6.1 交易列表加载
- **API**: GET `/api/transactions?book_id={bookId}&page=1&page_size=50`
- **参数**：
  - `book_id`：当前用户默认账本ID
  - `page`：固定为 1
  - `page_size`：固定为 50
  - 可选：`tag` 按标签名称筛选

### 6.2 交易详情加载
- **API**: GET `/api/transactions/{id}`
- **加载时机**：进入编辑页时自动获取
- **数据映射**：
  - direction === 'in' → type = 'income'
  - direction === 'out' → type = 'expense'
  - tags 字段需要 JSON 解析后匹配标签ID

### 6.3 关联数据加载
交易列表和编辑页会同时加载以下数据：
- **账户列表**: GET `/api/accounts?book_id={bookId}`
- **分类列表**: GET `/api/categories?book_id={bookId}`
- **标签列表**: GET `/api/tags?book_id={bookId}`

## 七、总结

交易页面的核心操作流程：

```
入口 (菜单/FAB/按钮)
    ↓
交易列表页 (TransactionsPage)
    ├── 筛选 → 按标签过滤
    ├── 批量选择 → 删除/退款
    └── 点击单条 → 编辑页
    ↓
编辑/新增页 (TransactionFormPage)
    ├── 切换类型 → 支出/收入
    ├── 填写表单 → 金额/账户/分类/日期/标签/备注
    ├── 验证 → 必填项检查
    └── 提交 → 新增(PUT) 或 编辑(PATCH)
    ↓
返回列表页
```

所有操作均围绕交易记录的全生命周期：查看 → 筛选 → 新增 → 编辑 → 删除/退款。
